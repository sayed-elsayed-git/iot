Packet Tracer - Blink an LED Using Blockly

Objectives

Part 1: Examine a Prebuilt Blockly Program

Part 2: Control a RGB LED using Blockly

Background

Blockly is a visual programming language that lets users create programs by connecting blocks that represent different logic language structures, rather than by writing the actual code. Blockly runs within a web browser and can translate the visually created program as JavaScript, PHP, or Python. In this Packet Tracer activity, you will use Blockly to examine Blockly programming and to control an LED.

Scenario

Using Blockly programming to control an IoT object LED. In this activity, Cisco Packet Tracer is used as it provides Blockly support with IoT objects.

Instructions

Part 1: Examine a Prebuilt Blockly Program

In this part, you will access the Cisco Packet Tracer program and examine LED control using Blockly programming.

Step 1: Examine the usage of the LED.

a.     Click LED to open the configuration window.

b.     Review the specification for the LED. This information is necessary when programming the LED later in this activity. Leave the window open for reference.

Step 2: Examine a prebuilt Blockly program.

a.     Click MCU to open the configuration window.

b.     Click the Programming tab to display the prebuilt Blockly program.

c.     Click Run.

Step 3: Change from digital to analog.

The LED specifications indicate that analogWrite can be used to adjust the device brightness. In this step, you will change from digital to analog and observe the change in LED brightness as you change the values in the program.

a.     In the Programming tab of the MCU, click Pin Access group to see all the options.

b.     Select analogWrite to replace digitalWrite in the Blockly program. Keep all other values the same.

c.     Now change the values of the first and second analogWrite blocks and observe the different LED brightness levels after restarting the program. For example, change the values to 100 and 1023 to see the different LED brightness levels.

d.     Click Stop to stop the program.

Part 2: Control a RGB LED using Blockly

In this part, you will use Blockly to control an RGB LED. An RGB can display different colors with the combination of red, green, and blue.

Step 1: Add an MCU board and an RGB LED.

In this step, you will add another MCU board and a RGB LED into the workspace.

a.     Copy MCU on the workspace. With the MCU highlighted, copy (Ctrl + C) and paste (Ctrl +V) it onto the workspace. Double click the display name of the copied board and rename MCU(1) to MCU-RGB.

b.     Click Components and select Actuators and add a RGB LED to the workspace. Rename the RGB LED as LED-RGB.

Step 2: Connect MCU board to the RGB LED.

a.     Click the RGB LED to view its specifications. Review the provided information so you can connect and program the LED correctly. Note different pin inputs represent different colors. Leave the window open.

b.     Click the Connections category, select three IoT Custom Cables to link MCU-RGB and LED-RGB.

According to the RGB LED specifications:

A0: Red

A1: Green

A2: Blue

Therefore, MCU-RGB D0 port is connected to RGB-LED A0 port for red LED color.

Note: On the MCU board, make sure you are connected to the digital ports (D0, D1, and D2). On the LED side, the analog ports (A0, A1, A2) are used.

c.     Repeat the same procedure for green and blue LED colors.

Green:  MCU-RGB D1 to LED-RGB A1

Blue:     MCU-RGB D2 to LED-RGB A2

Step 3: Modify the Blockly program.

In this step, you will program the RGB LED by modifying the program used for a single-colored LED.

a.     Click the MCU-RGB. Click Programming. You should see the program for controlling the single-colored LED modified in the previous part.

b.     Expand Pin Access group, and add two more pinMode blocks to set three slots as OUTPUT (from MCU-RGB to send a signal to RGB-LED).
